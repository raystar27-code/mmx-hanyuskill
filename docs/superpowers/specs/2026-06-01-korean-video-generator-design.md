# 中韩朗读高亮视频生成器 Skill 设计文档

本文档定义了 `korean-video-generator` Skill 的设计规格。该 Skill 旨在为 Hermes 智能体提供一个全自动的工具：输入中文文本或文档，自动翻译为韩语，并生成带有逐句高亮变色效果的 1080x1920 竖屏朗读视频。

## 1. 业务目标与核心功能

*   **输入**：
    *   **单语模式**：一段纯中文文本，或一个纯中文文本文件。
    *   **双语模式**：已提供中韩对照的文本或文件（格式为 `中文 | 韩文` 逐行对应，或交替行形式）。
*   **处理流程**：
    1.  **翻译（可选）**：如果输入为纯中文，则使用 MiniMax 语言大模型将其翻译为韩文。如果输入已经是中韩双语对照格式，则直接跳过翻译步骤。
    2.  **分句对齐**：将中文原文和韩语译文按照句子粒度进行切分与一一对齐。
    3.  **TTS 语音合成**：针对每一句韩文，调用 MiniMax 高品质 TTS 接口生成对应的单句音频文件。
    4.  **动态高亮图片生成**：使用 Python Pillow 库，生成 N 张 1080x1920 竖屏图片（N 为句子总数）。第 i 张图片中，第 i 句（当前朗读句）的韩文呈亮金色、中文呈纯白色；非当前朗读的所有其他句子呈清晰易读的明亮浅灰色。
    5.  **音视频合成**：调用系统 `ffmpeg` 命令行工具，将每句的高亮图片与对应的 TTS 音频合成为单句 MP4 视频，最后将所有单句 MP4 串联拼接成最终的完整视频。
*   **输出**：一个高质量 MP4 格式的竖屏视频文件，视频长度与 TTS 朗读总时长一致。

## 3. 详细技术实现方案

### 3.1 翻译与文本对齐 (Translation & Alignment)
*   **输入检测与处理模式**：
    *   **自适应解析**：脚本会优先检测输入内容。如果内容中包含特定分隔符（如每行格式为 `中文句子 | 韩文句子`）或者显式指定 `--bilingual` 参数，则**直接读取并进行对齐**，不调用翻译大模型。
    *   **翻译引擎**：如果输入为纯中文，调用 MiniMax Chat 完成翻译。使用系统 Prompt 约束，要求翻译后输出特定的分隔格式（如 `中文句子 | 韩文句子`），以便于程序统一解析。
*   **解析规则**：
    *   读取解析后的数据，提取出 `(中文句子, 韩文句子)` 的对齐元组列表。
    *   例如：`[("今天天气真好。", "오늘 날씨가 정말 좋네요."), ("我们出去玩吧。", "우리 놀러 가요.")]`

### 3.2 语音合成 (MiniMax TTS)
*   **配音接口**：调用 MiniMax 的 Speech API（通过 CLI 或 HTTP 请求直接调用）。
*   **配音角色**：支持两种高质量内置发音：
    *   `female` (标准韩语女声)
    *   `male` (标准韩语男声)
*   **单句合成**：逐句请求 API 生成单个 `.mp3` 或 `.wav` 音频，并自动从文件属性获取其精确的时长（以秒为单位）。

### 3.3 竖屏图片排版与渲染 (Pillow Image Rendering)
*   **规格**：1080px (宽) × 1920px (高)，分辨率 72 DPI。
*   **背景色**：纯黑色 (`#000000`)。
*   **字体**：使用 `resources/font.ttf` 字体文件，字号统一设为适合大屏阅读的尺寸（例如韩文 48px，中文 36px）。
*   **配色机制**：
    *   **当前高亮句**：
        *   **韩文**：亮金黄色 (`#FFD700`)，加粗。
        *   **中文**：纯白色 (`#FFFFFF`)。
    *   **非高亮句（其他所有句子）**：
        *   **中韩文**：明亮浅灰色 (`#B0B0B0`)。此对比度在黑底上十分清晰，保证用户能看清全文。
*   **布局与自动换行**：
    *   文字整体垂直居中对齐。
    *   实现自动折行算法，若单行文本宽度超出 960px（留出左右边距），则自动进行换行排版。

### 3.4 视频合成 (FFmpeg Processing)
*   **单句视频合成**：
    对于第 i 句，使用第 i 张高亮图片 `img_i.png` 和第 i 个音频 `audio_i.mp3` 生成 `segment_i.mp4`：
    ```bash
    ffmpeg -y -loop 1 -i img_i.png -i audio_i.mp3 -c:v libx264 -tune stillimage -c:a aac -b:a 192k -pix_fmt yuv420p -shortest segment_i.mp4
    ```
*   **视频串联**：
    生成拼接描述文件 `concat_list.txt`，并执行串联命令：
    ```bash
    ffmpeg -y -f concat -safe 0 -i concat_list.txt -c copy final_output.mp4
    ```

## 4. 依赖项与约束

1.  **系统工具**：系统必须安装有 `ffmpeg` 命令行工具且可用。
2.  **Python 库**：
    *   `Pillow` (用于图片排版绘制)
    *   `requests` (用于直接请求 MiniMax API)
3.  **API 凭证**：需要配置 `MINIMAX_API_KEY`（或直接使用用户提供的 API 密钥）。

---

## 5. 验证与测试标准

1.  **翻译准确性**：检查生成的对齐文本，确保句子数量和中文韩文严格一一对应。
2.  **图片可读性**：高亮句和置灰句过渡自然，文字在纯黑底色下清晰可见，无韩文乱码。
3.  **音画同步**：最终 MP4 中，声音播放到某句时，视频画面中对应的句子立刻变更为金黄色，播放完毕后恢复为浅灰色。
